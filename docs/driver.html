BISICLES driver executable

BISICLES' driver executable is used to run time-dependent ice sheet simulations. At the start of a simulation, it carries out the following operations
  1. Compute initial thickness, topography, temperature, damage and basal friction coeffcient data over the whole domain on the base level (L = 0) mesh. This data might be loaded from a file through the LevelData interface, or retrieved from the CISM interface, or computed through the Python interface
  2. Solve the stress-balance equations to find the velocity field on the l = 0 mesh
  3. Generate a new mesh with L + 1 levels on which some regions are more finely resolved according to the refinement criteria
  4. Compute the thickness, topography, temperature, damage and basal friction coeffcient data on the L + 1 mesh
  5. Solve the stress-balance equations to find the velocity field on the L + 1 mesh
  6. Write a plot file, including the thickness, topography, surface elevation, veleocity etc
  7. if L has not reached its maximum value, set L = L + 1 and repeat 3-7
It then runs the following operations until the simulation has completed
  1. Write a check-point file, if required
  2. If time t > simulation time, exit, else continue
  3. Compute surface fluxes (accumulation and melting) given the geometry and velocity field
  4. Compute horizontal thickness fluxes given the geometry, velocity field and surface fluxes
  5. Evolve the thickness field for a time determined by the CFL condition
  6. Solve the stress-balance equations to find a new velocity field
  7. Write a plot file, if required
  8. Update the mesh according to the refinement criteria
  9. If needed, solve the stress-balance equations to find a new velocity field
  10. repeat from step 1
